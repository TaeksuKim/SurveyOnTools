## 1. Executive Summary

- **7종 비교로 확장:** 기존 6종 체제에 모델 비종속적 도구의 벤치마크로서 **Cline**이 추가되어, 보다 포괄적인 에이전트 생태계 비교가 완성되었다.
- **범용 4종은 "인프라 성숙도와 유연성"이 강점**(메모리·압축·하니스·샌드박싱·체크포인트). Claude Code가 가장 포괄적인 하니스를 제공하며, Codex는 무인 실행·정책엔진, Antigravity는 Knowledge Base·역할분담 병렬·네이티브 브라우저, Cline은 압도적인 모델 비종속성과 SDK 확장성을 자랑한다.
- **특수 3종은 "뾰족한 특화 기능"이 강점**: OpenClaw=메시징 UI 및 스킬 생태계, oh-my-openagent=토큰최적 하니스 및 9개 전문 에이전트 멀티모델 라우팅, Hermes=자기개선 학습 루프 및 다채널 세션 핸드오프.
- **단일 승자 없음** — 용도(인프라·안전 vs 벤더중립·장기 자율 실행 vs 특수 채널)에 따라 적합 도구가 갈린다.
- **차세대 설계 핵심**: 계층형 메모리, 연합 에이전트 프로토콜, 프로그레시브 신뢰 모델, 자기개선 폐루프 표준화 등 15가지 핵심 혁신 과제가 도출되었다.

## 3. 범용 4종 심층 코멘트

### 3.1 Claude Code — "최소 스캐폴딩, 최대 하니스"
코드의 ~1.6%만 AI 결정 로직이고 98.4%가 운영 인프라라는 *harness-first* 철학이 정체성이다. 5계층 컨텍스트 압축, 27종 lifecycle hooks, 7단계 권한 모드, OS 샌드박싱, 그리고 변경 전 자동 스냅샷 + `/rewind` 등 운영 성숙도가 뛰어나다. 서브에이전트는 격리 컨텍스트(sidechain)로 동작하며 메모리는 버전 관리 가능한 파일 기반이다. 
**한계**: 멀티모델 라우팅은 자사 계열 중심(◐)이며, 메신저 다채널이나 세션 핸드오프 기능은 부족하다. [1][4][5][6]

### 3.2 Codex CLI — "표준 + 무인 실행 + 정책 엔진"
`AGENTS.md`(Linux Foundation 표준화)와 2-모델 비동기 메모리로 컨텍스트를 효율적으로 관리한다. 보안 하니스가 특히 단단하여 Rust `execpolicy` 기반 Starlark 정책 엔진과 macOS Seatbelt/Linux Landlock OS 격리를 제공한다. Goal Mode를 통한 장기 목표 수행과 `codex exec` 무인 실행에 적합하다. 
**한계**: 32KiB 상한의 컨텍스트 압축, 제한적인 라이프사이클 훅 기능. [7][9][10][11]

### 3.3 Antigravity — "Agent-first IDE + Knowledge Base + 역할분담 병렬"
Gemini 3.1 Pro 기반으로 CLI, IDE, Desktop, SDK 4-surface를 통합 지원한다. 과거 작업의 컨텍스트를 축적하는 Knowledge Base와 Mission Control 기반 병렬 에이전트(Manager/Writer/Critic/Tester) 오케스트레이션이 돋보인다. 내장 브라우저, Terminal Sandboxing(`nsjail`/`sandbox-exec`) 및 CLI의 강력한 `/rewind`, `/fork` 체크포인트를 지원한다.
**한계**: 멀티모델 라우팅이 자사 모델 중심(◐)이며 완전한 개방형 스킬 생태계 확장은 상대적으로 제한적이다. [13][16][17][18]

### 3.4 Cline — "완전한 모델 비종속성 + 확장성"
가장 포괄적인 모델 비종속성(Anthropic, OpenAI, Gemini, Local Ollama 등)을 제공하며 SDK 5계층 아키텍처 위에서 동작한다. Memory Bank 기반의 영구 메모리, Auto Compact 및 JIT 컨텍스트 관리 시스템을 채택했다. MCP Marketplace를 통한 플러그인 생태계와 Plan/Act 모드 분리가 강력하다.
**한계**: 빌트인 네이티브 멀티/서브 에이전트 오케스트레이션 부재, 자체 OS 샌드박스 미제공(◐). [19][22][23][24]

## 4. 특수목적 3종 심층 코멘트

### 4.1 OpenClaw — 로컬 기반 확장형 메시징 에이전트
WhatsApp, Telegram, Slack 등 메시징 플랫폼 네이티브 통합과 `SKILL.md` 기반의 성숙한 스킬 시스템(ClawHub)이 최대 강점이다. 브라우저, canvas, cron 등을 1급 타입드 도구로 노출한다. 
**한계**: 자율 실행 특성과 악성 스킬 패키지로 인한 보안 취약점이 보고되어 샌드박스(○) 강화 및 격리 환경 실행이 강력히 권고된다. 동적 메모리 갱신도 부분적이다. [26][27][32]

### 4.2 oh-my-openagent (OmO) — 토큰 최적·멀티모델 오케스트레이터
IntentGate를 통해 사용자의 의도를 사전 분류하고 카테고리별로 최적 모델을 라우팅하는 기술이 독보적이다. Sisyphus, Prometheus, Atlas 등 9개 전문 에이전트로 구성된 Team Mode 병렬 작업과 LSP+AST 기반 정밀 코드 인텔리전스를 제공한다.
**한계**: 오케스트레이션 구조의 설정 복잡도가 높으며, 다채널/크로스 디바이스 사용성이나 자체 샌드박싱 부재. [33][36][37]

### 4.3 Hermes Agent — 자가학습·멀티플랫폼 연속성
작업 성공 패턴을 스스로 추출해 스킬로 자동 승격시키는 **Closed Learning Loop**가 가장 진보적인 차별점이다. 3-Tier 메모리(Short/Long/Procedural), `/handoff`를 통한 무중단 크로스 디바이스 세션 이전, 내장 스케줄링(cron)을 제공한다.
**한계**: 로컬 기반 자동 학습에 따른 보안 및 프롬프트 인젝션 관리 필요, OS 샌드박스 네이티브 지원 미흡. [38][39][45]

## 5. 차세대 도구 설계 아이디어

에이전트 생태계의 향후 발전을 위해 15가지 핵심 아이디어를 도출했다.
1. **투명 파일 메모리 + 비동기 2-모델 관리**: 파일 기반 메모리의 투명성과 비동기 갱신을 결합.
2. **자기개선 폐루프(Closed Learning Loop) 표준화**: 경험을 스킬로 자동 승격.
3. **계층형 메모리 아키텍처 통합**: Working, Episodic, Semantic 3계층 분리 및 FTS5 검색 연동.
4. **다계층 컨텍스트 압축 + 컨텍스트 증류**: 단순 요약이 아닌 결정/제약조건의 구조적 보존.
5. **Harness-first + deny-first 보안 인프라**: 정책 엔진 및 커널 수준 샌드박스 기본 탑재.
6. **프로그레시브 신뢰 모델 (Progressive Trust Escalation)**: 신뢰도에 따른 동적 권한 확대.
7. **작업별 멀티모델 라우팅 및 의도 자동 분류 (IntentGate)**: 벤더 독립적 작업 효율화.
8. **연합 에이전트 프로토콜 (Federated Agent Protocol)**: 이종 에이전트 프레임워크 간 협업 표준.
9. **역할 분담형 병렬 멀티에이전트**: 작성-검증 역할 분리(Critic/Tester).
10. **검증 가능한 산출물(Artifacts) 도입**: 런타임 검증을 위한 중간 산출물 시각화.
11. **체크포인트/롤백 1급 안전망 필수화**: 코드와 대화 히스토리의 완전한 스냅샷 롤백.
12. **CI-Native 헤드리스 표준**: 비대화형 실행 및 구조화 출력 통합.
13. **cron 무인 실행 + 다채널 핸드오프**: 플랫폼 경계를 넘나드는 장기 자율 실행.
14. **SDK 기반 통합 플러그인 프레임워크**: 확장 가능한 에이전트 런타임.
15. **팀/공유 메모리 배포 표준화**: 개인정보를 배제한 지식 패키징 및 동기화 (최대 미개척 기회).
